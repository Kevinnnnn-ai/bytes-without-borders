## II - Features
- **Micro-lesson shelf**: 12 lessons—9 articles and 3 interactive quizzes—evenly split across the three topics (data privacy, tech literacy, digital inclusion), registered in [./docs/data/lessons.json](docs/data/lessons.json) and filterable by topic on the lessons hub.
- **Interactive quizzes**: [./docs/js/quiz.js](docs/js/quiz.js) renders question/choices/explanation quizzes from `docs/data/quiz-*.json`, with a progress bar and retry.
- **Progressive enhancement**: every page is fully readable and navigable with JavaScript (JS) off; JS-only chrome (mobile nav reveal, hub filter, back-to-top, share button, contact form) is created or revealed by JS so no-JS visitors never see dead controls.
- **Internationalization (i18n)**: inline English plus `data-i18n` keys resolved against `docs/locales/<code>.json`—switching applies in place with no reload. Spanish is live site-wide, with two fully translated lesson copies under [./docs/lessons/es/](docs/lessons/es/) cross-linked via `hreflang` alternates and `data-alt-*` attributes.
- **Per-page security headers**: every page carries a Content-Security-Policy (CSP) meta locked to `'self'` plus sha256 hashes of its exact inline scripts, regenerated by [./tests/tools/update_head_meta.py](tests/tools/update_head_meta.py).
- **Self-contained at runtime**: no external requests—fonts are self-hosted variable woff2 files in `docs/assets/fonts/`; dark scheme, reduced-motion, and print styles included.
- **Theatrical layer**: site-wide ambient system (aurora mesh, paper grain, animated gradient ink, torn-paper edges, cross-page view transitions) with per-tier set-pieces — envelope hero, scroll-drawn flight path and stamp marquee on the home page; 3D postcard tilt, postmarks and magnetic chips on the hub; stamp-confetti and a count-up finale in quizzes — all in [./docs/css/main.css](docs/css/main.css) plus one dependency-free [./docs/js/theatre.js](docs/js/theatre.js), fully degrading for no-JS, reduced-motion, print, and mobile.
- **Validation suite**: 37 pytest checks in [./tests/test_site.py](tests/test_site.py) verify required pages, exact-case internal links, JSON shapes, i18n key parity, translation cross-links, sitemap coverage, CSP hash drift, and per-page basics.
- **Continuous integration (CI)**: [./.github/workflows/tests.yml](.github/workflows/tests.yml) runs the same suite on every push and pull request as a non-blocking signal.

## III - Demonstration
Running the validation suite prints a one-line-per-test summary:

```text
$ python -m pytest tests -q
.....................................                                    [100%]
37 passed in 0.25s
```

Each pytest session also writes a plain-text report to `stdout/last-test-report.txt`:

```text
Bytes Without Borders site validation
collected: 37
failed: 0
exit status: 0
```

Site content is data-driven where it repeats—each lesson is one entry in `docs/data/lessons.json`:

```json

 "title": "Understanding passwords",
 "topic": "data-privacy",
 "summary": "Why one strong, unique password per account beats a clever one used everywhere.",
 "minutes": 4,
 "kind": "article"
}
```

## IV - Quick Start
```powershell
# 1. Clone the repository and move into it
git clone https://github.com/Kevinnnnn-ai/bytes-without-borders.git
cd bytes-without-borders

# 2. Create and activate a virtual environment named .env.local (validation only)
python -m venv .env.local
.\.env.local\Scripts\Activate.ps1

# 3. Install the test dependency and validate the site
pip install -r requirements.txt
python -m pytest tests -q

# 4. Preview the site locally
python -m http.server 8000 -d docs
```

**Note**: the preview server runs until you stop it (`Ctrl+C`) and sends no `Cache-Control` header—browsers may cache `main.css` aggressively between edits, so hard-refresh when verifying style changes. There is no build step; the files in `docs/` are the site.

## VI - Usage
All commands run from the **repository root** with the `.env.local` virtual environment active.

### Preview the site
Serves `docs/` at `http://localhost:8000`:

```powershell
python -m http.server 8000 -d docs
```

### Run the validation suite
Runs all 37 checks and writes `stdout/last-test-report.txt`:

```powershell
python -m pytest tests -q
```

### Regenerate security metas
Required after editing **any** inline `<script>` in `docs/`—a stale CSP hash silently drops that page to its no-JS fallback (`test_head_security_meta` catches drift):

```powershell
python tests\tools\update_head_meta.py
```

### Extend the site
Step-by-step recipes for adding an article, an interactive quiz, or a language live in [./superpowers/architecture.md](superpowers/architecture.md). In short: copy an existing page in `docs/lessons/`, register it in `docs/data/lessons.json` and the hub, add it to `docs/sitemap.xml`, and run the suite.

### Deploy
GitHub Pages branch publishing (repository settings → Pages → Deploy from a branch, `main`, `/docs`). Every push to `main` publishes `docs/` as-is; CI cannot block it, so treat a red `tests.yml` run as a broken production site. `docs/.nojekyll` disables Jekyll processing.

## VIII - Reference
### Project layout
```text
bytes-without-borders/
├─ docs/                         # the publishable site (GitHub Pages serves this folder)
│  ├─ index.html                 # home: hero, proof strip, featured lessons
│  ├─ about.html / get-involved.html / 404.html
│  ├─ lessons/
│  │  ├─ index.html              # lessons hub with topic filter
│  │  ├─ *.html                  # 9 articles + 3 quiz pages
│  │  └─ es/                     # Spanish translated page copies (pilot: 2 lessons)
│  ├─ data/                      # lessons.json registry + quiz-*.json question sets
│  ├─ locales/                   # en.json / es.json chrome + UI dictionaries
│  ├─ js/                        # main.js (enhancement), i18n.js (language), quiz.js (quizzes), theatre.js (ambient theatrics)
│  ├─ css/main.css               # all design tokens and styles
│  ├─ assets/                    # favicon, logo, self-hosted fonts
│  ├─ robots.txt / sitemap.xml   # sitemap is hand-maintained, suite-enforced
│  └─ .nojekyll                  # disables Jekyll processing on Pages
├─ tests/
│  ├─ test_site.py               # 37 validation checks
│  ├─ conftest.py                # writes stdout/last-test-report.txt per session
│  └─ tools/update_head_meta.py  # regenerates per-page CSP hashes
├─ superpowers/                  # project docs (architecture, specs, plans)—not published
├─ stdout/                       # test artifacts (gitignored contents)
├─ .github/workflows/tests.yml   # CI: pytest on every push/pull request
└─ requirements.txt              # pytest pin
```

### Key entry points
- **[./docs/index.html](docs/index.html)**: the site's front door; every page is a complete HTML document with duplicated shared chrome (header/nav/footer)—there are no partials and no build step.
- **[./docs/js/i18n.js](docs/js/i18n.js)**: dictionary-swap localization; exposes `window.bwbDict`, fires `bwb:langchange`, and navigates to translated page copies via `body[data-alt-<code>]`.
- **[./tests/test_site.py](tests/test_site.py)**: the source of truth for site invariants—run it before and after any change.

### How it works at a glance
1. Each page in `docs/` is standalone: relative URLs only, a head-inline script that adds a `js` class to `<html>`, and a hash-pinned CSP meta.
2. JavaScript is enhancement only—`main.js` reveals hidden-by-default chrome, `quiz.js` renders quizzes from JSON, `i18n.js` swaps `data-i18n` text against a locale dictionary.
3. Pushing to `main` publishes `docs/` directly through GitHub Pages branch publishing; the pytest suite (locally and in CI) guards links, shapes, parity, and coverage.

### External
- [GitHub Pages branch publishing](https://docs.github.com/en/pages): hosting—no Actions deploy step.
- Runtime: none—no external fonts, scripts, styles, or requests.
